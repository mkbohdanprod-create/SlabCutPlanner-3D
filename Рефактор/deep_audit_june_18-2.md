## 🟡 Технічний Борг (Низький Пріоритет)

Наступні пункти **не є критичними блокерами**. Їх варто вирішувати інкрементально (по мірі роботи з відповідними файлами), а не виділяти під них окремий спринт рефакторингу.

1. **i18n у TypeScript Types:** Використання українських рядків як дискримінантів (`type DetailType = 'Стільниця' | 'Мийка'`). Коли виникне потреба в багатомовності (англійська версія) — перевести на `enum`.
2. **Bundle Size (2 MB):** Головний чанк все ще досить великий через парсери (DXF/Approval). Lazy import для `FormsPanel` має сенс, але лише якщо реальні користувачі скаржаться на швидкість завантаження.
3. **Тести слайсів та UI:** Поточних Snapshot-тестів достатньо для безпеки ядра. Тести компонентів (React Testing Library) — варто впроваджувати поступово.
4. **Файли-монстри (God Objects):**

> [!WARNING]
> 7 файлів перевищують 50 KB. Хоча їх розмір суттєво зменшився після першої фази рефакторингу, вони все ще ускладнюють читабельність та продуктивність розробки в цих модулях. **Ми не рефакторимо їх абстрактно "на майбутнє", а розбиваємо ТІЛЬКИ коли заходимо туди для виконання продуктової фічі.**

### Задача #1: Фаски та зрізи під кутом у 3D
**Опис:** Промалювання фасок та зрізів під кутом у 3D (можливо, окремим модулем).
| Аспект | Технічне рішення |
|---|---|
| **Архітектура** | Окремий мікрододаток (React + R3F) — правильне рішення для ізоляції складної 3D-математики. |
| **Data Contract** | Розширити `DetailPart` полями: `chamfers: { edge: string, type: 'round'\|'bevel', radius: number }[]`, `bevelAngle: number`. |
| **Обмін даними** | JSON через `postMessage` (якщо iframe) або URL query params / LocalStorage. |
| **Що рефакторити перед фічею** | Саме **перед цією задачею** доведеться розбити `Viewer3D.tsx`, виділивши `TexturedPart` як незалежний перевикористовуваний компонент, щоб він міг приймати параметри фасок. |

### Задача #2: Онікс який просвітлюється (дві текстури)
**Опис:** Додати кнопку підсвітки, яка перемикає/змішує два фото текстури оніксу.
| Аспект | Технічне рішення |
|---|---|
| **Модель даних** | Додати до `SlabInstance` поля: `photoBacklit?: string`, `materialKind?: 'standard' \| 'onyx'`. |
| **UI** | Кнопка-перемикач у `Sidebar3D.tsx` (toggle `backlightEnabled`). Додати `backlightEnabled: boolean` у `EditorUISlice`. |
| **3D Рендер** | У `Viewer3D` (в `TexturedPart`) додати другий матеріал з `emissiveMap` або використовувати ShaderMaterial для змішування двох текстур. |
| **Готовність** | Висока. Додатковий рефакторинг не потрібен, лише розширення стейту та 3D-матеріалу. |

### Задача #4: Набір команд для MES/WMS/APS
**Опис:** Кнопка, яка генерує виробничі команди і перекидає на платформу виробництва.

|---|---|
