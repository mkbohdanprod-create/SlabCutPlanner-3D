# 📦 Пакет передачі для Antigravity — два блоки з ісходника

Тут два готові блоки, витягнуті зі старої (Electron) версії та підготовлені під поточну веб-архітектуру (React 19 + Zustand 5 slices + immer + Supabase). Передай цей пакет в Antigravity разом із посиланням на локальний проєкт.

## Порядок виконання

**Спочатку — ЕКОНОМІКА (легша, чистіша, найвищий ROI). Потім — APPROVAL.**

Загальні правила для обох блоків:
- Працювати **строго послідовно**, `npx tsc --noEmit` після кожного кроку.
- Коміти **атомарні** (по одному логічному кроку).
- **Не пушити на GitHub**, поки користувач не підтвердить, що локально все працює (білий екран ловиться лише в браузері — обов'язково Ctrl+Shift+R + консоль F12, бо `tsc` цей клас помилок не бачить).

## Блок 2 — Імпорт бланків узгодження (Задача #5)

```
approval/
├── INTEGRATION_APPROVAL.md      ← ГОЛОВНА ІНСТРУКЦІЯ. Читати першою.
└── approvalImport.CANONICAL.ts  ← найновіша версія (з OCR), готова під GitHub-конфіг
```

**Стан:** канонічний файл (build 20260617, з OCR) замінює застарілу копію в репо (build 20260610, без OCR). API та типи сумісні (перевірено). Потрібно: додати залежність `tesseract.js`, замінити файл, перевірити на бланках. **Декомпозиція god-object'а (3566 рядків) — окрема пізніша сесія, не зараз.**

## Чому саме так (коротко)

Обидва блоки існували працюючими в Electron-ісходнику, але загубились/застаріли при міграції на веб:
- економіку **не перенесли** взагалі;
- approvalImport у репо — **старіший на 7 днів** і без OCR.

Тому це не розробка з нуля, а **відновлення вже зробленого** в новій архітектурі. Перенось у такому порядку, тестуй локально, і лише після підтвердження користувача переноси на GitHub.
